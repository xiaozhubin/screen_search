为什么我在 GitHub Pages 部署后无法连通本地？

这是由于浏览器极其严格的安全屏障带来的常见问题

当你把此控制台托管在 GitHub Pages 上（通过 https://<name>.github.io 访问）时，由于控制台处于 HTTPS 安全协议 中，而您的 Screenpipe 本地服务通常是在 HTTP（localhost:3030） 运行：

  • 混合内容阻挡 (Mixed Content)：现代浏览器为了防止窃听，默认会直接封死 HTTPS 页面向 HTTP 发送的异步 AJAX 请求 (Fetch/XHR)。
  • 跨域拦截 (CORS)：如果 Screenpipe 的底层 API 没有配置显式的跨域允许白名单，请求会卡在 Preflight 握手期。

推荐的 4 个通关姿势

方案一 / 最方便：本地保存运行（极力推荐）

不需要服务器，不需要任何部署。直接在浏览器上按下 Ctrl + S（或右键另存为），将当前网页保存为本地的 index.html 文件。双击在浏览器里打开它，由于没有域限制，它能无障碍跨越沙箱访问本地 API。

方案二 / 修改浏览器混合安全设置

如果坚持在 GitHub 线上直接运行，可以点击浏览器地址栏最左侧的 网站设置（一把小锁图标），找到 “不安全的内容”(Insecure content)，并将其修改为 “允许”，刷新网页即可。

方案三 / 使用 ngrok 等反向代理

将本地服务映射为安全的 HTTPS 地址（如运行 ngrok http 3030 映射出公网 SSL），然后将生成的公网 HTTPS 链接手动填入本站顶部的 HOST 配置框。

方案四 / 本地启动简易 HTTP 服务

在本地有 index.html 的路径上运行 npx serve，启动一个轻量级的 HTTP 本地服务器访问它，从而完美绕开 HTTPS 的严格域名安全限制。
